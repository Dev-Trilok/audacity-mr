4 4 Nyquist is a LISP like language integrated into Audacity. Nyquist ओड्यासिटीमध्ये समाकलित भाषेसारखी LISP आहे.
5 5 It was originally written to process audio. हे मूळतः ध्वनीवर प्रक्रिया करण्यासाठी लिहिले गेले होते.
6 6 Nyquist now also has the ability to control Audacity, similar to how Macros work, but with greater flexibility. नायक्विस्टमध्ये आता मॅक्रोस कसे कार्य करतात यासारखेच ओड्यासिटी नियंत्रित करण्याची क्षमता देखील आहे, परंतु मोठ्या लवचिकतेसह.

16 16 Comparison of Ordinary Macros to Nyquist Macros ऑर्डिनरी मॅक्रोची तुलना Nyquist मॅक्रोशी आहे
17 17 Ordinary Macros execute a fixed sequence of commands in a fixed order. सामान्य मॅक्रो एक निश्चित क्रमाने आदेशांचा निश्चित क्रम चालवतात.
18 18 By contrast, the Nyquist language is a full language from the LISP family. That makes it more powerful and also more complex to use. याउलट, Nyquist भाषा ही LISP कुटुंबातील संपूर्ण भाषा आहे. हे अधिक सामर्थ्यवान आणि वापरण्यास अधिक जटिल बनवते.
19 19 Nyquist has commands for logic, loops and conditionally executing code. Nyquist कडे लॉजिक, लूप्स आणि सशर्त कोड कार्यान्वित करण्याच्या आज्ञा आहेत.
20 20 Nyquist supports streams and string formatting functions. Nyquist प्रवाह आणि स्ट्रिंग स्वरूपन कार्ये समर्थित करते.
21 21 Nyquist supports data structures, and is especially strong in handling lists. Nyquist डेटा स्ट्रक्चर्सना समर्थन देते, आणि खासकरुन याद्या हाताळणीत मजबूत आहे.
22 22 Nyquist, in Audacity, can prompt for parameters and then use those parameters in its code. ऑडीसिटी मध्ये Nyquist पॅरामीटर्ससाठी प्रॉमप्ट करू शकते आणि नंतर त्या कोडमध्ये त्या पॅरामीटर्सचा वापर करू शकतो.
23 23 Nyquist can be used independently of Audacity. ऑयडिसिटी स्वतंत्रपणे ओड्यासिटी वापरली जाऊ शकते.
24 24 When used from within Audacity, it gains an additional command, AUD-DO. जेव्हा ओड्यासिटी मधून वापरले जाते तेव्हा ते एयूडी-डीओ अतिरिक्त आदेश प्राप्त करते.
25 25 AUD-DO AUD-do
26 26 AUD-DO is a function that extends Nyquist's capabailities. एयूडी-डीओ एक फंक्शन आहे जे नेक्विस्टच्या क्षमता वाढवते.
27 27 AUD-DO makes it possible to use all Audacity Macro scripting commands from within Nyquist. एयूडी-डीओ नेकीक्झीट मधून सर्व ओड्यासिटी मॅक्रो स्क्रिप्टिंग आज्ञा वापरणे शक्य करते.
28 28 The full list of scripting commands and their parameters is provided at the page Scripting Reference. स्क्रिप्टिंग आज्ञा्स आणि त्यांचे पॅरामीटर्सची पूर्ण यादी पृष्ठ स्क्रिप्टिंग संदर्भ पृष्ठावर प्रदान केली आहे.
29 29 The scripting commands are passed to AUD-DO as strings. स्क्रिप्टिंग आज्ञा एआरडी-डीओला स्ट्रिंग म्हणून दिले जातात.
30 30 A very similar approach is used for Audacity Python Scripting. ओड्यासिटी पायथन स्क्रिप्टिंगसाठी अगदी समान दृष्टीकोन वापरला जातो.
31 31 Commands must be created as strings and passed to a function to process. आज्ञा्स स्ट्रिंग म्हणून तयार केल्या पाहिजेत आणि प्रक्रियेसाठी फंक्शनला पुरविल्या पाहिजेत.
32 32 One advantage of Nyquist Macros over Python Scripting is that Nyquist is already built in to Audacity. पायथन स्क्रिप्टिंगपेक्षा नेयक्विस्ट मॅक्रोजचा एक फायदा असा आहे की नायक्विस्ट आधीपासून ओड्यासिटीमध्ये अंगभूत आहे.
33 33 Using AUD-DO AUD-DO वापरणे
34 34 Your Nyquist Script will need to have "type tool". आपल्या Nyquist स्क्रिप्टमध्ये "टाइप साधन" असणे आवश्यक आहे.
35 35 AUD-DO is not available for other types. इतर प्रकारच्या एडीडी-डीओ उपलब्ध नाहीत.
36 36 ;type tool टाइप साधन
37 37 The following command in a nyquist script would set the first label in a project to 'foo' Nyquist स्क्रिप्ट मध्ये खालील आज्ञा प्रोजेक्ट मधील पहिले लेबल 'foo' वर सेट करते.
38 38 (AUD-DO "SetLabel: Text=\"Foo\"") (एडीओ-डीओ "सेट लेबल: मजकूर = \" फू \ "")
39 39 The text string after AUD-DO is passed to the scripting engine to process. एयूडी-डीओ नंतर मजकूर स्ट्रिंग प्रक्रियेसाठी स्क्रिप्टिंग इंजिनला दिली जाते.
40 40 A complete example of a Nyquist Script using AUD-DO that you can enter at the Tools Nyquist Prompt is given below: आपण साधन्स नायक्विस्ट प्रॉम्प्ट वर प्रविष्ट करू शकता असे एयूडी-डीओ वापरुन Nyquist स्क्रिप्टचे संपूर्ण उदाहरण खाली दिले आहे:

57 57 Differences to normal Nyquist Scripting सामान्य Nyquist स्क्रिप्टिंग मध्ये फरक
58 58 The '$type' setting in Nyquist sets what capabilities Nyquist will have for that Nyquist script. Nyquist मधील '$ प्रकार' सेटिंग Nyquist मध्ये त्या Nyquist स्क्रिप्टसाठी कोणती क्षमता आहे हे सेट करते.
59 59 For example, $type process and $type analyze scripts are given the *TRACK* variable. उदाहरणार्थ, $ टाइप प्रक्रिया आणि $ टाइप विश्लेषण स्क्रिप्ट्सला * गीतपट्टा * व्हेरिएबल दिले जाते.
60 60 Process scripts, when applied, will be called repeatedly, once for each channel to process. प्रक्रिया स्क्रिप्ट, लागू केल्यावर, प्रत्येक वाहिनीवर प्रक्रिया करण्यासाठी एकदा, पुन्हा कॉल केल्या जातील.
61 61 Another use of the '$type' setting is to set which menu in Audacity a script will be placed in. '$ प्रकार' सेटिंगचा आणखी एक उपयोग ऑडिटी मध्ये कोणत्या मेनूमध्ये स्क्रिप्ट ठेवला जाईल हे सेट करणे आहे.
62 62 It can occasionally be useful to place a process, generator or analyze Nyquist script in the tools menu, even though its functionality is one of the other types. साधने मेनूमध्ये एक प्रक्रिया ठेवणे, जनरेटर तयार करणे किंवा त्याचे विश्लेषण करणे कधीकधी उपयुक्त ठरू शकते, जरी त्याची कार्यक्षमता इतर प्रकारांपैकी एक असते.
63 63 To do that you can use two arguments to the '$type' setting. असे करण्यासाठी आपण 'प्रकार' सेटिंगमध्ये दोन वितर्क वापरू शकता.
64 64 You can create nyquist plug-ins with for the Tools menu using any of the following: आपण खालीलपैकी कोणतेही एक वापरुन साधन्स मेनूसह nyquist प्लगइन तयार करू शकता:
65 65 $type tool $ प्रकार साधन
66 66 $type tool process tool प्रकारची साधन प्रक्रिया
67 67 $type tool generator tool साधन जनरेटर टाइप करा
68 68 $type tool analyze tool टाइप साधन विश्लेषण
69 69 The first argument to 'type' determines which Audacity menu the plug-in ends up in. 'टाइप' करण्यासाठी प्रथम वितर्क कोणत्या ओड्यासिटी मेनूमध्ये प्लग-इन समाप्त होतो हे निर्धारित करते.
70 70 All four examples are for the Tools menu. सर्व चार उदाहरणे साधन्स मेनूसाठी आहेत.
71 71 The qualifier process / generator / analyze determines how the plug-in actually behaves. पात्रता प्रक्रिया / जनरेटर / विश्लेषण हे प्लग-इन प्रत्यक्ष कसे वर्तन करते हे निर्धारित करते.
72 72 Only '$type tool' should be used with AUD-DO. AUD-DO सह केवळ '$ प्रकार साधन' वापरले जावे.

95 95 Click Removal Example काढण्याचे उदाहरण क्लिक करा
96 96 On the scripting page, for example, the parameters for the click removal command are documented: स्क्रिप्टिंग पृष्ठावर, उदाहरणार्थ, क्लिक रिमूव्हल आज्ञाचे मापदंड दस्तऐवजीकरण केलेले आहेत:
97 97 int Threshold, (default:200) इंट थ्रेशोल्ड, (पूर्वनिर्धारित: २००)
98 98 int Width, (default:20) इंट रुंदी, (पूर्वनिर्धारित: 20)
99 99 To invoke the same command in Nyquist, you need to pass a string to AUD-DO. Nyquist मध्ये समान आज्ञासाठी, आपल्याला AUD-DO कडे एक स्ट्रिंग पास करणे आवश्यक आहे.
100 100 For example: उदाहरणार्थ:
101 101 (AUD-DO "ClickRemoval: Threshold=200 Width=25") (एडीडी-डीओ "क्लिक रिमूव्हल: थ्रेशोल्ड = 200 रुंदी = 25")
102 102 Default values do not need to be specified, so a shorter version of the same command would be: पूर्वनिर्धारित मूल्य निर्दिष्ट करण्याची आवश्यकता नाही, म्हणून समान आदेशाची छोटी आवृत्ती अशी असेल:
103 103 (AUD-DO "ClickRemoval: Width=25") (एडीडी-डीओ "क्लिक रिमूव्हल: रुंदी = 25")
104 104 GetInfo Example गेटइन्फो उदाहरण
105 105 The AUD-DO "GetInfo" Scripting commands provide results as strings, and offers three different result formats. एयूडी-डीओ "गेटइंफो" स्क्रिप्टिंग आज्ञा्स स्ट्रिंग म्हणून परिणाम प्रदान करतात आणि तीन भिन्न परिणाम स्वरूप ऑफर करतात.
106 106 It is usually easiest, in Nyquist, to ask for the results of GetInfo commands as strings in LISP format एलआयएसपी स्वरूपात स्ट्रिंग म्हणून गेटइन्फो आज्ञाचा निकाल विचारणे, नेयक्विस्टमध्ये सहसा सोपे असते.
