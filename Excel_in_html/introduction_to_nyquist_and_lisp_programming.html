4 4 Nyquist was written by Roger B. Dannenberg and was intended to be used as a complete programming language for audio synthesis and analysis, with support for MIDI, audio recording and playback, file I/O, object-oriented programming, profiling, debugging and more. Audacity uses only a subset of Nyquist’s functionality, allowing you to take simple Nyquist functions and use them to process audio data. Audacity doesn’t include any support for debugging Nyquist code, so if you are trying to write a complicated plug-in, you may find it easier to get the full version of Nyquist and develop there, then turn it into an Audacity plug-in. Nyquist is available from the Carnegie Mellon University Computer Music Project. नायक्विस्ट रॉजर बी. डॅन्नेनबर्ग यांनी लिहिलेले होते आणि ध्वनी संश्लेषण आणि विश्लेषणासाठी एमआयडीआय, ध्वनी रेकॉर्डिंग आणि प्लेबॅक, धारिका आय / ओ, ऑब्जेक्ट-ओरिएंटेड प्रोग्रामिंग, प्रोधारिकािंग, डीबगिंग आणि बरेच काही यासाठी एक संपूर्ण प्रोग्रामिंग भाषा म्हणून वापरली जाण्याची इच्छा होती . ओड्यासिटी केवळ Nyquist च्या कार्यक्षमतेचा उपसंच वापरते, आपल्याला साधी Nyquist कार्ये करण्याची आणि ध्वनी डेटावर प्रक्रिया करण्यासाठी त्यांचा वापर करण्याची परवानगी देते. ऑडीसिटीमध्ये एनक्यूइस्ट कोड डीबग करण्यासाठी कोणतेही समर्थन समाविष्ट नाही, म्हणून जर आपण एक गुंतागुंतीचे प्लग-इन लिहिण्याचा प्रयत्न करीत असाल तर आपल्याला Nyquist ची संपूर्ण आवृत्ती मिळवणे आणि तेथे विकसित करणे सुलभ वाटेल, तर त्यास ओड्यासिटी प्लग-इनमध्ये रूपांतरित करा. . कार्केगी मेलॉन युनिव्हर्सिटी संगणक संगीत प्रकल्पातून Nyquist उपलब्ध आहे.

16 16 Nyquist allows you to use the Nyquist programming language to write your own plug-in effects for Audacity. Unlike VST and LADSPA plug-ins, Nyquist plug-ins can be written using an ordinary text editor and don’t need to be compiled. ऑयडीसिटी आपल्याला स्वतःचा प्लग-इन प्रभाव लिहिण्यासाठी Nyquist प्रोग्रामिंग भाषा वापरण्याची परवानगी देतो. व्हीएसटी आणि एलएडीएसपीए प्लगइनच्या विपरीत, न्यक्विस्ट प्लगइन एक सामान्य मजकूर संपादक वापरून लिहिले जाऊ शकतात आणि त्यास कंपाईल करण्याची आवश्यकता नाही.
17 17 Nyquist supports both a Lisp syntax and a more conventional syntax called SAL. Audacity versions prior to 1.3.8 only support Lisp, but the current Audacity 2.0 series supports both Lisp and SAL. 21 21 Lisp लिस्प
22 22 Nyquist is based on Lisp. If you have programmed in Lisp before, you can skim this section or go directly to the next page: Programming in Nyquist . Otherwise, here’s an extremely brief introduction to Lisp: Nyquist लिस्पवर आधारित आहे. जर आपण आधी लिस्पमध्ये प्रोग्राम केलेले असेल तर आपण हा विभाग वगळू शकता किंवा पुढील पृष्ठावर जाऊ शकताः प्रोग्रामिंग नायक्विस्टमध्ये. अन्यथा, इथे लिस्पची अत्यंत थोडक्यात माहिती आहेः
23 23 In Lisp (and therefore Nyquist), everything is an S-Expression, which is just a list of tokens (words) separated by whitespace and enclosed in parentheses. The name of the function is always the first token in an S-Expression, and all of the other tokens are arguments to this function. Here’s a simple example: लिस्प (आणि म्हणून न्यक्विस्ट) मध्ये, प्रत्येक गोष्ट एस-एक्सप्रेशन असते, जी केवळ स्पेसद्वारे विभक्त आणि कंसात बंद केलेल्या टोकनची (शब्द) यादी आहे. एस-एक्सप्रेशन मधील फंक्शनचे नाव नेहमीच प्रथम टोकन असते आणि इतर सर्व टोकन या फंक्शनचे वितर्क असतात. येथे एक साधे उदाहरण आहेः
24 24 (setf area (* 3.14159 (expt radius 2))) (सेटफ क्षेत्र (* 3.14159 (एक्सप्ट त्रिज्या 2)))
25 25 Let’s break down this example. The outermost S-expression has three members. The first one, setf, is the name of the function (it stands for set-field). setf is used to assign a value to a variable. (There are other similar functions, like set and setq, but setf is the most powerful, so it’s the one we’ll use in our examples.) After setf comes area, which is the name of the variable we’re going to set. Next comes the value to assign to this variable, which in this case is another S-expression. चला या उदाहरणाचे खंडन करूया. बाहेरील एस-अभिव्यक्तीचे तीन सदस्य आहेत. पहिला सेटफ, फंक्शनचे नाव आहे (ते सेट-फील्डसाठी वापरले जाते). व्हेरिएबलला व्हॅल्यू देण्यासाठी सेटफचा वापर केला जातो. (इतर समान कार्ये आहेत जसे की सेट आणि सेटेक, परंतु सेटफ सर्वात शक्तिशाली आहे, म्हणूनच आपण आपल्या उदाहरणांमध्ये वापरू.) सेटफचे क्षेत्रफळ आल्यानंतर आपण बदलणार्या व्हेरिएबलचे नाव आहे. . पुढे या व्हेरिएबलला असाइन करण्यासाठी व्हॅल्यू मिळेल, जे या प्रकरणात आणखी एक एस-एक्सप्रेशन आहे.
26 26 Lisp doesn’t have any special operators for Math functions – they’re all functions like everything else, using prefix notation, where the name of the function (or operator) comes before its arguments. So instead of 3*7 for the product of 3 and 7, in Lisp you would write (* 3 7). In Nyquist, the expt (exponent) function raises its first argument to the power of the second argument. Therefore (* 3.14159 (expt radius 2)) means 3.14159 times the square of radius, or the formula for the area of a circle. लिस्पकडे मॅथ फंक्शन्ससाठी कोणतेही विशेष ऑपरेटर नसतात - ते प्रत्येकाच्या सारख्या सर्व फंक्शन्स असतात, प्रत्यय नोटेशन वापरुन, जेथे फंक्शनचे नाव (किंवा ऑपरेटर) त्याच्या वितर्कांपूर्वी येते. तर 3 आणि 7 च्या उत्पादनाऐवजी लिस्पमध्ये (* 3 7) लिहा. Nyquist मध्ये, एक्सप्ट (घातांक) फंक्शन दुसर्‍या वितर्क च्या सामर्थ्यावर आपला पहिला युक्तिवाद दर्शवितो. म्हणून (* 14.१14१15 ((एक्सप्ट रेडियस २)) म्हणजे त्रिज्याच्या चौकोनाच्या पटापेक्षा जास्त किंवा वर्तुळाच्या क्षेत्राचे सूत्र.
27 27 Rather than typing in this full expression every time, let’s define a function for the area of the circle, that we can call every time we need it: प्रत्येक वेळी या पूर्ण अभिव्यक्तीवर टाइप करण्याऐवजी आपण वर्तुळाच्या क्षेत्रासाठी फंक्शन परिभाषित करू या, ज्यावेळेस आम्हाला जेव्हा आवश्यक असेल तेव्हा आम्ही कॉल करू:
28 28 (defun circlearea (radius) (सर्कलिया (त्रिज्या)
29 29 (* 3.14159 (expt radius 2))) (* 3.14159 (एक्सपोर्ट त्रिज्या 2%)))
30 30 The defun function is used to define a new function. The first argument is the name of the function, in this case circlearea. The second argument is a list of arguments to the function to be defined – this is one of the few cases where you have an S-expression that is not interpreted as a function call. Finally the last expression is the value of the function. Now if we want to compute the area of a circle of radius r, we just need to compute: Defun फंक्शन नवीन फंक्शन परिभाषित करण्यासाठी वापरले जाते. प्रथम युक्तिवाद फंक्शनचे नाव आहे, या प्रकरणात मंडळामध्ये. दुसरा युक्तिवाद म्हणजे कार्य करण्यासाठी परिभाषित केलेल्या वितर्कांची यादी - हे अशा काही प्रकरणांपैकी एक आहे जेथे आपल्याकडे एस-एक्सप्रेशन आहे ज्याचे फंक्शन कॉल म्हणून अर्थ लावले जात नाही. शेवटी अंतिम अभिव्यक्ती फंक्शनचे मूल्य आहे. जर आपल्याला त्रिज्या r च्या वर्तुळाचे क्षेत्रफळ मोजायचे असेल तर आपल्याला फक्त गणना करणे आवश्यक आहेः
31 31 (setf area (circlearea r)) (सेटफ क्षेत्र (वर्तुळ क्षेत्र आर))
32 32 An S-expression is just a representation of a list. Lisp uses lists to represent just about everything (the name LISP comes from LISt Processing language), so it’s helpful to know how to manipulate lists. Let’s start by assigning a list of numbers to a variable. You can’t quite do this: एस-अभिव्यक्ती ही केवळ सूचीचे प्रतिनिधित्व असते. लिस्प सर्वकाही प्रतिनिधित्व करण्यासाठी याद्या वापरतात (एलआयएसपी हे नाव एलआयएसटी प्रक्रियेच्या भाषेतून आले आहे), म्हणून याद्या कशा हाताळता येतील हे जाणून घेणे उपयुक्त ठरेल. चला व्हेरिएबलला नंबरची यादी देऊन सुरूवात करू. आपण हे जोरदार करू शकत नाही:
33 33 (setf mylist (1 2 3 4 5)) (सेटफ मायलिस्ट (1 2 3 4 5))
34 34 <-- <-
35 35 error! चूक!
36 36 The reason this doesn’t work is that whenever Nyquist sees an S-expression, it tries to evaluate it as a function unless you tell it otherwise. Since there’s no function named “1” that takes arguments (2 3 4 5), this will generate an error. To tell Lisp that you want to treat an S-expression literally, and not to evaluate it as a function, you quote it. In Nyquist, you can quote a list by putting a single quotation mark before it, like this: हे कार्य करत नाही याचे कारण असे आहे की जेव्हा जेव्हा नेक्विस्ट एस-एक्सप्रेशन पाहतो तेव्हा आपण कार्ये म्हणून त्याचे मूल्यांकन करण्याचा प्रयत्न करतो जोपर्यंत आपण त्यास अन्यथा सांगत नाही. “1” नावाचे कोणतेही कार्य नसलेले आर्ग्युमेंट्स घेतात (2 3 4 5), यामुळे एक त्रुटी निर्माण होईल. लिस्पला सांगण्यासाठी की तुम्हाला एस-अभिव्यक्ती शब्दशः करायची आहे आणि त्याचे कार्य म्हणून मूल्यमापन करायचे नाही, आपण ते उद्धृत करा. Nyquist मध्ये, आपण यापूर्वी एकच कोटेशन चिन्ह ठेवून सूचीचे कोट लावू शकता:
37 37 (setf mylist '(1 2 3 4 5)) (सेटफ मायलिस्ट '(1 2 3 4 5))
38 38 Nyquist also provides a list function that you can use to construct lists – this is useful if some of the elements of the list are functions: Nyquist एक यादी कार्य देखील प्रदान करते जे आपण याद्या तयार करण्यासाठी वापरू शकता - जर सूचीतील काही घटक कार्ये असतील तर हे उपयुक्त आहे:
39 39 (setf mylist (list 1 2 3 4 (sqrt 25))) (सेटफ मायलिस्ट (यादी 1 2 3 4 (स्क्वेअर 25)))
40 40 To get things off of a list, you can use the first and rest functions. (Traditionally, these were called car and cdr, respectively, butfirst and rest are much easier to remember. Both sets of names are supported in Nyquist.) The output of (first mylist) is 1, and the output of (rest mylist) is the list (2 3 4 5). So the second element of the list is (first (rest mylist)). गोष्टी यादीतून काढून टाकण्यासाठी आपण प्रथम आणि उर्वरित कार्ये वापरू शकता. (पारंपारिकपणे, त्यांना अनुक्रमे कार आणि सीडीआर म्हटले गेले, परंतु बुफर्स्ट आणि विश्रांती लक्षात ठेवणे खूप सोपे आहे. दोन्ही नावे सेट्स नेक्विस्टमध्ये समर्थित आहेत.) (प्रथम मायलिस्ट) आउटपुट 1 आहे, आणि उर्वरित मायलिस्टचे आउटपुट आहे यादी (2 3 4 5). तर सूचीचा दुसरा घटक म्हणजे (प्रथम (बाकीची माझी यादी)).
